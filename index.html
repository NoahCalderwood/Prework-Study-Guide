HTML

  • The body element represents the visible content shown to the user.
  • LISTS
    1. ul is Unordered list
    2. ol is Ordered list or numbered
    3. li is a list item
  • link needs rel or relationship and href or URL to what is being linked.
  • img and script use src = as their path.

CSS


  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git


Git Started


  • mkdir [folder-name]: creates a folder in current location
  • cd: change directory. cd .. is used to go back one directory.
  • touch: creates a file

  • git checkout -b branch-name: creates a new branch and switches to it
  • git init [project-name]: creates a new local repository and Git initializes a new repository inside it.
  • git clone [project url]: Downloads a project with the entire history from the remote repository.

Git Information


  • git status: checks what branch we are currently on
  • git branch -a: lists all branches
  • li: lists all directories and files

Git Workflow


  • git add -A: stages all files in the working branch
  • git pull origin main: ensures branch is up to date
  • git commit -m "message": commits te changes staged with a message
  • git push origin branch-name: pushes all changes to Github

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.

  • Functions have two parts to operate.
    1. Declaring the function: uses the function command. Give function a name. Write code to be executed
    2. Calling the function: uses the functions declare name to execute code.

  • array example: const fruits = ['Banana', 'Apple', 'Strawberry', 'Tomato']
  • for loops: set up format like "for (let x=0; x < "length"; x++)"
  • if, else if, else: conditional logic used to execute different code based on input
